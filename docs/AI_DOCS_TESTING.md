# Test-Driven Development (TDD) & Testing Strategy

> **Core Principle**: TDD is **non-negotiable**. Write tests before implementation.

---

## The TDD Workflow

### 1. Red: Write Failing Test

```python
# Backend example: test_selector.py
def test_selector_returns_highest_priority_exercise():
    """Given a list of exercises, should return the highest priority one."""
    exercises = [
        Exercise(name="Squats", priority=5),
        Exercise(name="Deadlifts", priority=10),
        Exercise(name="Bench Press", priority=3),
    ]
    
    selector = ExerciseSelector()
    result = selector.select_highest(exercises)
    
    assert result.name == "Deadlifts"
```

### 2. Green: Implement Code to Pass Test

```python
# app/services/selector.py
class ExerciseSelector:
    def select_highest(self, exercises: list[Exercise]) -> Exercise:
        return max(exercises, key=lambda ex: ex.priority)
```

### 3. Refactor: Improve Code

```python
class ExerciseSelector:
    def select_highest(self, exercises: Sequence[Exercise]) -> Exercise:
        """Select the exercise with the highest priority.
        
        Args:
            exercises: Non-empty sequence of exercises.
            
        Returns:
            The exercise with the highest priority value.
            
        Raises:
            ValueError: If exercises is empty.
        """
        if not exercises:
            raise ValueError("Cannot select from empty exercise list")
        return max(exercises, key=lambda ex: ex.priority)
```

### Test-First Benefits

- **Clearer requirements**: Writing tests clarifies what code should do
- **Immediate feedback**: Red/green provides fast feedback
- **Better design**: Testing-first leads to loosely-coupled code
- **Regression prevention**: Tests catch breaking changes early
- **Documentation**: Tests serve as living documentation

### Unit Test Pattern

```python
import pytest
from app.services import ExerciseSelector
from app.models import Exercise

class TestExerciseSelector:
    """Tests for the ExerciseSelector service."""
    
    @pytest.fixture
    def selector(self) -> ExerciseSelector:
        """Fixture: Create a selector instance."""
        return ExerciseSelector()
    
    @pytest.fixture
    def exercises(self) -> list[Exercise]:
        """Fixture: Create test exercises."""
        return [
            Exercise(name="Squats", priority=5, difficulty="intermediate"),
            Exercise(name="Deadlifts", priority=10, difficulty="advanced"),
            Exercise(name="Bench Press", priority=3, difficulty="beginner"),
        ]
    
    def test_select_highest_priority(
        self, selector: ExerciseSelector, exercises: list[Exercise]
    ) -> None:
        """Given exercises with different priorities, should return highest."""
        result = selector.select_highest(exercises)
        assert result.name == "Deadlifts"
        assert result.priority == 10
    
    def test_select_highest_raises_on_empty_list(
        self, selector: ExerciseSelector
    ) -> None:
        """Given empty list, should raise ValueError."""
        with pytest.raises(ValueError, match="Cannot select from empty"):
            selector.select_highest([])
    
    def test_select_highest_with_single_exercise(
        self, selector: ExerciseSelector
    ) -> None:
        """Given single exercise, should return it."""
        exercises = [Exercise(name="Squats", priority=5)]
        result = selector.select_highest(exercises)
        assert result.name == "Squats"
```

### Async Test Pattern

```python
import pytest
from app.services import PlanGenerator

class TestPlanGenerator:
    """Tests for the PlanGenerator service."""
    
    @pytest.fixture
    async def generator(self) -> PlanGenerator:
        """Fixture: Create a generator instance."""
        loader = MockExerciseLoader()
        return PlanGenerator(loader)
    
    @pytest.mark.asyncio
    async def test_generate_plan_returns_valid_structure(

        request = PlanRequest(
            program_id="strength_101",
            athlete_level="intermediate",
            duration_weeks=12,
        )

        assert result.program_id == "strength_101"

    @pytest.mark.asyncio

```

### Using Fixtures Effectively

```python
import pytest
from unittest.mock import AsyncMock, MagicMock

class TestPlanValidator:

    def validator(self, mock_schema_store, mock_exercise_loader):
```

### Snapshot Testing (Complex Outputs)

For complex JSON outputs (like generated plans), use snapshot tests:

```python
import pytest
from syrupy import SnapshotAssertion

class TestPlanGeneration:
    """Tests using snapshot assertions."""
    
    @pytest.mark.asyncio
    async def test_generated_plan_structure(
        self,
        generator: PlanGenerator,
        snapshot: SnapshotAssertion,
    ) -> None:
        """Generated plan should match expected structure."""
        request = PlanRequest(
            program_id="strength_101",
            athlete_level="intermediate",
            duration_weeks=4,
        )
        plan = await generator.generate_plan(request)
        
        # First run: creates snapshot
        # Subsequent runs: compare against snapshot
        assert plan.model_dump() == snapshot
```

Run with: `pytest --snapshot-update` to update snapshots after intentional changes.

### Property-Based Testing

Use Hypothesis to generate test cases and find edge cases:

```python
from hypothesis import given, strategies as st
import pytest

class TestExercisePriority:
    """Property-based tests for exercise priority."""
    
    @given(
        st.integers(min_value=1, max_value=100),
        st.integers(min_value=1, max_value=100),
    )
    def test_priority_comparison_is_consistent(self, p1: int, p2: int) -> None:
        """Priority comparison should be consistent for any input."""
        ex1 = Exercise(name="A", priority=p1)
        ex2 = Exercise(name="B", priority=p2)
        
        # Test consistency
        if p1 > p2:
            assert ex1.priority > ex2.priority
        elif p1 < p2:
            assert ex1.priority < ex2.priority
        else:
            assert ex1.priority == ex2.priority
```

### API Endpoint Tests

```python
import pytest
from httpx import AsyncClient
from app.main import app

class TestPlanEndpoints:
    """Integration tests for plan endpoints."""
    
    @pytest.fixture
    async def client(self) -> AsyncClient:
        """Create async HTTP client."""
        async with AsyncClient(app=app, base_url="http://test") as client:
            yield client
    
    @pytest.mark.asyncio
    async def test_create_plan_endpoint(self, client: AsyncClient) -> None:
        """POST /v1/plans should create a plan."""
        request_data = {
            "program_id": "strength_101",
            "athlete_level": "intermediate",
            "duration_weeks": 12,
        }
        
        response = await client.post("/v1/plans", json=request_data)
        
        assert response.status_code == 201
        data = response.json()
        assert "id" in data
        assert data["program_id"] == "strength_101"
        assert len(data["weeks"]) == 12
    
    @pytest.mark.asyncio
    async def test_create_plan_validation_error(
        self, client: AsyncClient
    ) -> None:
        """POST /v1/plans with invalid data should return 400."""
        request_data = {
            "program_id": "",  # Empty
            "athlete_level": "invalid",  # Invalid enum
            "duration_weeks": 0,  # Invalid (must be > 0)
        }
        
        response = await client.post("/v1/plans", json=request_data)
        
        assert response.status_code == 400
        error = response.json()
        assert "detail" in error
```

### Component Test Pattern

```typescript
import { render, screen, fireEvent } from "@testing-library/react";
import { describe, it, expect, vi } from "vitest";
import { SessionCard } from "@/components/plan/SessionCard";

describe("SessionCard Component", () => {
```

### Hook Test Pattern

```typescript
import { renderHook, waitFor, act } from "@testing-library/react";
import { describe, it, expect, vi, beforeEach } from "vitest";
import { useGetPlan } from "@/hooks/useGetPlan";
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import React from "react";

describe("useGetPlan Hook", () => {
  let queryClient: QueryClient;

  beforeEach(() => {
    queryClient = new QueryClient();
  });

  it("should fetch plan data", async () => {
    const wrapper = ({ children }: { children: React.ReactNode }) =>
      React.createElement(QueryClientProvider, { client: queryClient }, children);

    const { result } = renderHook(() => useGetPlan("plan-123"), { wrapper });

    // Initially loading
    expect(result.current.isLoading).toBe(true);

    // Wait for data
    await waitFor(() => {
      expect(result.current.isLoading).toBe(false);
    });

    expect(result.current.data).toBeDefined();
    expect(result.current.data?.id).toBe("plan-123");
  });

  it("should handle errors gracefully", async () => {
    // Mock a failed request
    const mockQueryClient = new QueryClient({
      defaultOptions: {
        queries: {
          retry: false,
        },
      },
    });

    const wrapper = ({ children }: { children: React.ReactNode }) =>
      React.createElement(QueryClientProvider, { client: mockQueryClient }, children);

    const { result } = renderHook(() => useGetPlan("invalid"), { wrapper });

    await waitFor(() => {
      expect(result.current.error).toBeDefined();
    });

    expect(result.current.data).toBeUndefined();
  });
});
```

### Form Test Pattern

```typescript
import { render, screen, fireEvent, waitFor } from "@testing-library/react";
import userEvent from "@testing-library/user-event";
import { describe, it, expect, vi } from "vitest";
import { PlanForm } from "@/components/forms/PlanForm";

    // Fill form
    await user.selectOptions(screen.getByRole("combobox", { name: /program/i }), "strength-101");
    await user.selectOptions(screen.getByRole("combobox", { name: /level/i }), "intermediate");

    // Submit
    await user.click(screen.getByRole("button", { name: /generate/i }));

    // Verify callback
    await waitFor(() => {
      expect(onSuccess).toHaveBeenCalled();
    });
  });

  it("should show validation errors", async () => {
    const user = userEvent.setup();

    render(<PlanForm onSuccess={vi.fn()} />);

    // Submit without filling required fields
    await user.click(screen.getByRole("button", { name: /generate/i }));

    // Check errors appear
    await waitFor(() => {
      expect(screen.getByText(/program is required/i)).toBeInTheDocument();
    });
  });
});
```

## Shared Testing Practices

### Mocking External Services

```python
# Python
from unittest.mock import AsyncMock, MagicMock

# Mock async function
mock_loader = AsyncMock()
mock_loader.load_exercises.return_value = [...]

# Verify calls
mock_loader.load_exercises.assert_called_once_with("program-1")
```

```typescript
// TypeScript
import { vi } from "vitest";

// Mock function
const mockFetch = vi.fn();
vi.stubGlobal("fetch", mockFetch);

mockFetch.mockResolvedValueOnce({ json: async () => ({ id: "123" }) });

// Verify calls
expect(mockFetch).toHaveBeenCalledWith("/api/plans", expect.objectContaining({ method: "POST" }));
```

### Testing Error Scenarios

Always test both success and failure paths:

```python
# ✅ GOOD - Test both happy path and errors
def test_validation_with_valid_input():
    result = validate({"name": "Valid"})
    assert result is True

def test_validation_with_invalid_input():
    with pytest.raises(ValidationError):
        validate({"name": ""})

def test_validation_with_missing_required_field():
    with pytest.raises(ValidationError, match="required"):
        validate({})
```

## Checklist: Before Pushing Code

- [ ] **Test written first** (red → green → refactor)
- [ ] **All tests pass** locally
- [ ] **Minimum 80% coverage** for the feature
- [ ] **Edge cases tested** (empty lists, null values, 0, negative numbers, etc.)
- [ ] **Error scenarios tested** (invalid inputs, network failures, etc.)
- [ ] **No skipped tests** (`skip`, `xtest`, `.only`)
- [ ] **No hardcoded test data** in non-fixture code
- [ ] **Mocks used correctly** (verify behavior, not implementation details)
- [ ] **Async tests properly awaited** (async/await, promises)
- [ ] **Test names clearly describe what they test**
