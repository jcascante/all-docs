# Python Backend - Copilot Instructions

> Specialized guidelines for the FastAPI backend. Reference these when generating Python code in `apps/backend/`.

---

## Context

- **Location**: `apps/backend/`
- **Language**: Python 3.12+
- **Framework**: FastAPI with async/await
- **Type Checking**: mypy --strict (mandatory)
- **Linting**: Ruff (formatter + linter)
- **Testing**: pytest (unit, integration, property-based)
- **Minimum Coverage**: 80%

## Core Principles

### 1. Type Hints Are Mandatory
Every function, class, and method must have complete type hints.

```python
# ✅ GOOD
async def generate_plan(
    request: PlanRequest,
    service: PlanGenerator,
) -> GeneratedPlan:
    return await service.generate(request)

# ❌ BAD - Missing type hints
async def generate_plan(request, service):
    return await service.generate(request)
```

### 2. Test-First (TDD)
Write tests BEFORE implementing code.

```python
# Step 1: Write failing test
def test_selector_returns_highest_priority():
    exercises = [
        Exercise(name="A", priority=5),
        Exercise(name="B", priority=10),
    ]
    result = ExerciseSelector().select_highest(exercises)
    assert result.name == "B"

# Step 2: Implement code to pass test
class ExerciseSelector:
    def select_highest(self, exercises: list[Exercise]) -> Exercise:
        return max(exercises, key=lambda ex: ex.priority)
```

### 3. Async/Await Throughout
All I/O operations must be async. No blocking calls.

```python
# ✅ GOOD
async def get_exercises(self, program_id: str) -> list[Exercise]:
    return await self.db.fetch_exercises(program_id)

# ❌ BAD - Blocking call
def get_exercises(self, program_id: str) -> list[Exercise]:
    return self.db.fetch_exercises(program_id)  # Blocking!
```

### 4. Dependency Injection
Services should accept dependencies via constructor.

```python
# ✅ GOOD
class PlanGenerator:
    def __init__(self, loader: ExerciseLoader, validator: SchemaValidator):
        self.loader = loader
        self.validator = validator

# ❌ BAD - Creating dependencies inside
class PlanGenerator:
    def __init__(self):
        self.loader = ExerciseLoader()  # Hard to test!
        self.validator = SchemaValidator()
```

### 5. Custom Exceptions
Never use bare exceptions. Create domain-specific ones.

```python
# ✅ GOOD
class ValidationError(Exception):
    """Raised when validation fails."""
    pass

async def generate(self, request: PlanRequest) -> GeneratedPlan:
    try:
        validate(request)
    except ValidationError as e:
        raise HTTPException(status_code=400, detail=str(e))

# ❌ BAD
except Exception:  # Too broad!
    pass
```

## Code Structure

### Project Layout
```
apps/backend/
├── src/app/                 # FastAPI app
├── src/api/                 # Route handlers
├── src/models/              # Pydantic models
├── src/services/            # Business logic
├── src/core/                # Config, exceptions, logging
├── src/schemas/             # JSON Schema definitions
├── src/loaders/             # Data loading
└── tests/
    ├── unit/                # Unit tests
    ├── integration/         # Integration tests
    ├── api/                 # Endpoint tests
    └── conftest.py          # Fixtures
```

### Module Naming
- **Modules**: `lowercase_with_underscores` (e.g., `plan_generator.py`)
- **Classes**: `PascalCase` (e.g., `PlanGenerator`)
- **Functions**: `lowercase_with_underscores` (e.g., `generate_plan`)
- **Constants**: `UPPERCASE_WITH_UNDERSCORES` (e.g., `MAX_DURATION_WEEKS`)
- **Test Files**: `test_<module>.py` (e.g., `test_selector.py`)

### Import Organization
```python
# 1. Standard library
import asyncio
import json
from typing import Optional

# 2. Third-party
from pydantic import BaseModel
import httpx

# 3. Local
from app.models import ExerciseModel
from app.services import ExerciseService
```

## Patterns to Follow

### FastAPI Endpoint
```python
from fastapi import APIRouter, HTTPException
from typing import Annotated

router = APIRouter(prefix="/v1/plans", tags=["plans"])

@router.post("", response_model=GeneratedPlan, status_code=201)
async def create_plan(request: PlanRequest) -> GeneratedPlan:
    """Create a new training plan."""
    try:
        plan = await service.generate_plan(request)
        return plan
    except ValidationError as e:
        raise HTTPException(status_code=400, detail=str(e))
    except ExerciseNotFoundError:
        raise HTTPException(status_code=404, detail="Exercise not found")
```

### Service with Dependency Injection
```python
class PlanGenerator:
    def __init__(self, loader: ExerciseLoader, validator: SchemaValidator):
        self.loader = loader
        self.validator = validator
    
    async def generate_plan(self, request: PlanRequest) -> GeneratedPlan:
        """Generate a training plan."""
        exercises = await self.loader.load_exercises(request.program_id)
        self.validator.validate(request)
        # ... implementation
        return GeneratedPlan(...)
```

### Pydantic Model with Full Typing
```python
from pydantic import BaseModel, Field

class Exercise(BaseModel):
    """An exercise within a program."""
    name: str = Field(..., description="Exercise name")
    reps: int = Field(gt=0, description="Number of reps")
    sets: int = Field(gt=0, description="Number of sets")
    
    model_config = ConfigDict(extra="forbid")
```

### Test Fixture
```python
@pytest.fixture
def exercise_loader() -> ExerciseLoader:
    """Mock exercise loader fixture."""
    mock = AsyncMock()
    mock.load_exercises.return_value = [
        Exercise(name="Squats", id="ex1"),
        Exercise(name="Deadlifts", id="ex2"),
    ]
    return mock

@pytest.mark.asyncio
async def test_generate_plan(exercise_loader: ExerciseLoader) -> None:
    generator = PlanGenerator(exercise_loader, MockValidator())
    result = await generator.generate_plan(PlanRequest(...))
    assert result.id is not None
```

## Testing Requirements

### Test Naming
- Test functions: `test_<what_is_being_tested>`
- Test classes: `Test<ClassName>`
- Descriptive docstrings explaining what/why

```python
def test_selector_returns_highest_priority_when_multiple_exercises():
    """Given exercises with different priorities, should return highest."""
    # Test implementation
```

### Coverage Targets
- **Minimum**: 80% line coverage
- **Target**: 90%+ for business-critical code (plan generation, validation)
- Run: `pytest --cov=src --cov-report=html`

### Test Types
| Type | Tool | Use Case |
|------|------|----------|
| Unit | pytest | Test functions/classes in isolation |
| Integration | pytest | Test services + dependencies |
| Property-based | hypothesis | Generate test cases for edge cases |
| Snapshot | syrupy | Lock complex outputs (JSON) |
| API | pytest + httpx | Test endpoints end-to-end |

### Always Test
- ✅ Happy path (normal operation)
- ✅ Error cases (invalid input, missing data)
- ✅ Edge cases (empty lists, 0, negative numbers, None)
- ✅ Async operations (await, error handling)

## Error Handling

### Custom Exceptions
Define domain-specific exceptions in `src/core/exceptions.py`:

```python
class SchemaValidationError(Exception):
    """Raised when schema validation fails."""
    pass

class ExerciseNotFoundError(Exception):
    """Raised when an exercise is not found."""
    pass

class ProgramDefinitionError(Exception):
    """Raised when program definition is invalid."""
    pass
```

### In Endpoints
Convert exceptions to HTTP responses:

```python
try:
    plan = await service.generate(request)
except SchemaValidationError as e:
    raise HTTPException(status_code=400, detail=str(e))
except ExerciseNotFoundError:
    raise HTTPException(status_code=404, detail="Exercise not found")
except Exception as e:
    logger.error(f"Unexpected error: {e}", exc_info=True)
    raise HTTPException(status_code=500, detail="Internal server error")
```

### Never Do This
```python
# ❌ BAD
try:
    something()
except:  # Bare except - too broad!
    pass

# ❌ BAD
except Exception:  # Also too broad
    pass
```

## What NOT to Do

❌ Skip tests or write tests after code
❌ Use bare `except` or generic `Exception` catching
❌ Create dependencies inside classes (use injection instead)
❌ Mix async and sync code
❌ Hardcode configuration values
❌ Use `print()` instead of logging
❌ Have functions without type hints
❌ Ignore linting or type checking errors
